# 🚀 PhD Research Copilot 用户操作指南

> **博士科研全流程智能引擎** - 从文献阅读到论文开题的一站式科研助手

---

## 📖 简介

PhD Research Copilot 是一个专为博士生设计的全栈科研 AI 技能。它模拟顶尖博士生导师的思维模式，采用**"状态机架构"**，完美契合真实科研中"发散探索 → 归纳收敛 → 顶层建构 → 靶向深挖"的漏斗型认知路径。

### ✨ 核心能力

- 📚 **文献精读与速读** - 深度解析或快速筛选文献
- 🗂️ **文献库管理** - 自动整合文献元数据到解码矩阵
- 📝 **综述生成** - 系统性整合多篇文献，识别研究脉络
- 🎓 **开题框架构建** - 生成逻辑严密的博士论文大纲
- 📂 **课题管理** - 多课题并行管理与章节实施方案

---

## 🚀 快速开始

### 1. 准备工作

确保你的工作目录下存在以下结构（首次使用时会自动创建）：

```
/PhD_Workspace
  ├── /Literature_Notes/          # 文献笔记存放目录
  ├── /Research_Projects/         # 研究课题文件夹
  ├── Decoding_Matrix.md          # 文献解码整合表
  ├── Literature_Synthesis.md     # 阶段性文献综述
  └── PhD_Grand_Framework.md      # 博士论文课题大纲
```

### 2. 基本使用流程

```
阶段1: 探索期 (Current_Topic = Null)
  ↓
/速读 文献 → 快速筛选 → /入库 → 积累文献库
  ↓
/综述 → 识别研究脉络与GAP
  ↓
阶段2: 靶向期 (Current_Topic = 你的课题)
  ↓
/开题 → 生成博士论文框架
  ↓
/精读 核心文献 → 深度分析 → /入库
  ↓
/章节 第X章 → 生成实施方案
```

## 📋 指令详解

### 📌 `/速读 [文献全文/摘要/链接]`

**用途**：快速筛选文献，判断相关性

**适用场景**：
- 初次接触一个新领域，需要大量浏览文献
- 判断一篇文献是否值得深度精读
- 快速提取核心贡献和实验结果

**输出**：
- 简洁的速读笔记（1-2页）
- 保存至 `/PhD_Workspace/Literature_Notes/速读_[作者_年份_标题].md`

**示例**：
```
/速读 e:\文献\paper.pdf
```

---

### 📌 `/精读 [文献全文/摘要/链接]`

**用途**：深度解析核心文献，提取写作技巧和研究思路

**适用场景**：
- 核心参考文献的深度研究
- 学习顶会的写作技巧和论证策略
- 为自己的论文寻找理论支撑和方法借鉴

**输出**：
- 详细的精读笔记（5-10页）
- 包含：基本信息、核心概要、研究背景、方法论、实验分析、个人启发
- 保存至 `/PhD_Workspace/Literature_Notes/精读_[作者_年份_标题].md`

**示例**：
```
/精读 e:\文献\nature_paper.pdf
```

---

### 📌 `/入库 [可选：文献笔记路径]`

**用途**：将文献笔记的核心元数据提取到文献解码整合表

**说明**：
- 若未指定路径：自动读取最新的精读/速读笔记
- 若指定路径：读取指定的文献笔记文件

**提取字段**：
| 字段 | 含义 |
|------|------|
| WTD | 研究问题 (What To Do) |
| TAP | 方法路径 (Technical Approach) |
| ROF | 核心发现 (Results Of Findings) |
| GAP | 研究空白 (Gap) |
| RAT | 研究必要性 (Rationale) |
| POC | 我的批评点 (Point Of Critique) |
| RPP/MOP | 我的启发 (Research Potential Point) |

**输出**：
- 追加到 `/PhD_Workspace/Decoding_Matrix.md`

**示例**：
```
/入库                                    # 使用最新笔记
/入库 e:\PhD_Workspace\Literature_Notes\速读_Zhang_2024.md
```

---

### 📌 `/综述 [可选：文献范围]`

**用途**：系统性整合已读文献，生成研究脉络分析

**说明**：
- 若未指定范围：读取所有文献笔记
- 若指定范围：读取指定的文献子集

**输出结构**：
1. **宏观综合与主题脉络** - 核心问题、研究图景、术语表
2. **核心主题分类与深度解析** - 分类评述+单篇精解
3. **交叉对比与关系网络** - 演进路径、横向对比、观点交锋
4. **综合评述与未来展望** - 共识分歧、挑战瓶颈、5个未来方向

**输出**：
- 保存至 `/PhD_Workspace/Literature_Synthesis.md`

**使用建议**：每阅读5-10篇文献后执行一次

**示例**：
```
/综述
```

---

### 📌 `/开题 [可选：初步灵感/痛点]`

**用途**：生成具有强逻辑闭环的博士论文大纲

**说明**：
- 读取 `Literature_Synthesis.md`（如存在）
- 结合你的初步灵感生成大纲
- 自动设置 `Current_Topic`，进入【靶向模式】

**输出结构**（五部分硬核结构）：
1. **立题与破局** - 核心矛盾论与解决主线
2. **基石与冷启动** - 轻量化快速生成/求解
3. **多模态/多源信息融合** - 置信度感知联合增强
4. **物理第一性原理** - 物理先验驱动的残差学习
5. **工程价值闭环** - Scan-to-FEM自动化验证

**输出**：
- 保存至 `/PhD_Workspace/PhD_Grand_Framework.md`
- 附赠：小论文拆解发表策略图

**示例**：
```
/开题 我想研究基于深度学习的空间结构健康监测方法
```

---

### 📌 `/设题 [课题名称或大纲描述]`

**用途**：手动设置当前研究课题

**说明**：
- 直接设置 `Current_Topic`
- 跳过前期探索阶段，快速进入【靶向模式】

**示例**：
```

```

---

### 📌 `/课题 [操作类型] [参数]`

**用途**：管理多个研究课题

**操作类型**：

| 操作 | 功能 | 示例 |
|------|------|------|
| `创建` | 创建新课题文件夹和基础文件 | `/课题 创建 我的课题` |
| `更新` | 更新现有课题大纲 | `/课题 更新 我的课题` |
| `查看` | 查看指定课题完整信息 | `/课题 查看 我的课题` |
| `列表` | 列出所有已创建课题 | `/课题 列表` |
| `切换` | 切换当前活跃课题 | `/课题 切换 另一个课题` |

**创建课题时自动生成的文件**：
```
/Research_Projects/[课题名称]/
  ├── Grand_Framework.md      # 课题框架大纲
  ├── Current_Topic.md        # 当前课题状态
  ├── Literature_Review.md    # 课题相关文献综述
  └── /Papers/                # 小论文草稿
```

### 📌 `/章节 [章节编号] [章节标题]`

**用途**：基于博士论文大纲，生成具体章节的详细实施方案

**说明**：
- 读取 `PhD_Grand_Framework.md` 定位指定章节
- 将宏观目标拆解为可执行的研究步骤

**输出结构**：
1. 研究背景与痛点（3个具体痛点）
2. 核心创新与实施步骤（2个主要创新点）
3. 可行性分析（技术/理论/资源三维度）
4. 本章预期产出（理论/工程/论文三方面）
5. 创新点的三重体现（学术包装话术）
6. MVP下一步行动指令
7. 章节实施检查清单

**输出**：
- 保存至 `/Research_Projects/[课题]/Chapter_[编号]_[短标题].md`

**示例**：
```
/章节 第二章 基于物理感知的结构响应预测方法
```

## 📊 速读 vs 精读对比

| 维度 | `/速读` | `/精读` |
|------|---------|---------|
| **时间成本** | 5-10分钟 | 30-60分钟 |
| **内容深度** | 核心信息提取 | 深度分析+写作技巧 |
| **适用场景** | 初步筛选、大量浏览 | 核心文献深度研究 |
| **输出篇幅** | 1-2页 | 5-10页 |
| **文件名前缀** | `速读_` | `精读_` |
| **是否可入库** | ✅ 可以 | ✅ 可以 |
| **写作技巧分析** | ❌ 无 | ✅ 详细 |
| **个人启发深度** | 中等 | 深入 |

**建议策略**：
- 先用 `/速读` 快速筛选大量文献
- 对高相关性文献使用 `/精读` 深度研究
- 两者都可以通过 `/入库` 加入文献库

## 🎯 状态机说明

### 探索模式 (Current_Topic = Null)

- **特点**：AI 侧重于发散性思维，帮助寻找 Research Gap
- **速读/精读侧重**：提出3-5个潜在全新研究大方向
- **适用阶段**：博一、新领域探索、寻找课题

### 靶向模式 (Current_Topic = 具体课题)

- **特点**：AI 所有分析围绕课题进行"靶向收敛"
- **速读/精读侧重**：
  - 该文献方法能否解决课题痛点？
  - 其局限性是否证明课题的先进性？
  - 实验指标能否被借鉴？
- **适用阶段**：课题确定后、深入研究期

## 💡 使用技巧

### 1. 文献筛选策略

```
/速读 文献A
→ 相关性低 → 不入库，跳过
→ 相关性中 → /入库，保留备用
→ 相关性高 → /入库 + 标记后续精读
```

### 2. 定期综述习惯

建议每阅读 **5-10篇** 文献后执行 `/综述`，及时梳理研究脉络。

### 3. 课题切换注意

切换课题后，所有新的速读/精读将围绕新课题进行靶向分析。

### 4. 章节规划时机

在完成 `/开题` 或 `/课题 创建` 后，即可使用 `/章节` 规划具体实施。

## ⚠️ 常见问题

**Q1: `/速读` 和 `/精读` 的笔记都能 `/入库` 吗？**

✅ 是的，两者都可以通过 `/入库` 添加到文献解码整合表。速读适合快速筛选阶段的文献，精读适合核心文献。

**Q2: 如何查看已入库的文献？**

直接打开 `/PhD_Workspace/Decoding_Matrix.md` 文件查看。

**Q3: 可以管理多个课题吗？**

✅ 可以，使用 `/课题 创建` 创建多个课题，用 `/课题 切换` 在不同课题间切换。

**Q4: `/综述` 会读取哪些文献？**

默认读取 `/Literature_Notes/` 目录下所有 `.md` 文件（包括速读和精读笔记）。

**Q5: 如何修改已生成的笔记？**

直接编辑对应的 `.md` 文件即可。如需重新入库，使用 `/入库 [文件路径]`。
